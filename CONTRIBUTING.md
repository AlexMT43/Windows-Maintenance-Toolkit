# Contributing

1. Fork the repository.
2. Create a branch for the change.
3. Keep each module focused on one maintenance task.
4. Do not introduce destructive defaults.
5. Test on Windows 10 or Windows 11 with PowerShell 5.1 or later.
6. Document user-visible changes in `CHANGELOG.md`.
7. Open a pull request describing the risk and expected behavior.

Avoid hard-coded usernames, personal paths, credentials, tokens or machine-specific data.
